# 月历悬浮球（Calendar Float）

角色卡《命定之诗与黄昏之歌》的 DLC 项目。

本项目当前采用 **世界书runtime驱动**：

- 运行时主数据源是世界书
- 脚本运行时读取世界书中的 [`[节庆_索引]`]条目
- 脚本根据 MVU 变量、最近两层消息、用户消息、日期窗口来决定哪些条目命中
- 命中世界书条目时，通过 silent `injectprompt` + `should_scan` 触发酒馆扫描
- 提醒与书籍摘要由脚本直接做可见注入，UI 与触发系统共享同一份索引

---

## 当前架构

系统分成三层：

### 1. 前端脚本层

- 渲染悬浮月历 UI
- 读取世界时间 / MVU 变量 / 用户自定义事件
- 从 世界书 读取 runtime 索引并组装日历数据
- 在生成后与切换聊天时执行 runtime trigger scan
- 维护提醒摘要、书籍摘要等运行时变量
- 安装脚本自带的基础设施世界书条目

### 2. 世界书内容层

- 存放索引条目 [`[节庆_索引]`]
- 存放节庆介绍、书籍全文
- 为可扫描内容提供专属绿灯触发词
- 作为 UI 展示与 runtime 触发的统一数据源

### 3. 脚本托管基础设施条目层

- 变量列表条目
- `mvu_update` 规则条目
- 基础连通性检查与重装 / 卸载 / 导出

## 运行方式

### 触发流程

1. 读取世界书中的 [`[节庆_索引]`]
2. 读取当前MVU世界时间
3. 读取 MVU 变量
4. 读取最近两层聊天消息
5. 优先提取 `<gametxt>...</gametxt>` 内容
6. 若没有可用 `gametxt`，则回退到普通消息文本，并执行：
   - 去掉 `</think>` 之前的内容
   - 去掉 `<UpdateVariable>...</UpdateVariable>` 块
7. 根据统一触发映射判定节庆介绍 / 节庆提醒 / 节庆文本 / 书籍摘要 / 书籍全文是否命中
8. 若命中世界书扫描类内容，则 silent inject 专属 token，并开启 `should_scan`
9. 让 SillyTavern扫描命中对应绿灯条目
10. 若命中提醒或摘要，则脚本直接做可见 `injectprompt`

## 内容说明

在世界书里有`[节庆_索引]`、书籍全文(fulltxt)、节庆介绍

节庆提醒(reminder)和书籍摘要(abstract)在`[节庆_索引]`内部，选填

### 节庆提醒(reminder)

D0深度提示词
会在节庆的X天前/进行时插入提醒LLM
如果未提供自定义提醒文本，可回退到默认模板

### 节庆介绍(event)

介绍节庆相关活动，会在满足条件(当前时间和地点 or 聊天里提到节庆相关词)时触发

### 书籍(books)

分为摘要和全文

摘要:

- 不希望把整本书正文直接送进上下文时使用
- 触发方式: 满足`[节庆_索引]关键词`

全文:

- 文本支持MD格式，[newpage]会开新一页
- 默认触发方式: 只有用户消息包含`[节庆_OO节_文本_文本名]`

当前部分节庆和读物正文仍是占位文本(咕咕咕)，但已经保留好对应条目与触发逻辑，后续只需要补正文就行

## 索引维护

条目命名风格见 [`条目命名约定`]：

- 索引条目：`[节庆_索引]`
- 节庆介绍：`[节庆_OO节_介绍]`
- 节庆文本：`[节庆_OO节_文本_文本名]`

若同名节庆一年有多个日期档，例如“深蓝拍卖会”，建议追加日期后缀，例如：

- `[节庆_深蓝拍卖会_6-6_介绍]`
- `[节庆_深蓝拍卖会_12-12_介绍]`

## 索引结构

你可以继续像这样写节庆节点：

```yaml
节庆:
  - id: alliance_day
    名称: 盟约日
    开始: 01-08                                        // 格式:MM-DD
    结束: 01-08
    地点关键词:
      - 诺斯加德联盟
      - 白曜城
    event:
      世界书条目名称: "[节庆_盟约日_介绍]"
      关键字:
        - 盟约日
        - 霜狼盟约
        - 联盟建立
    reminder:
      开始前提醒天数: 5
      开启自定义提醒:                                   // 填false会使用缺省模板
        未开始: false
        进行中: "要写东西就自行填上内容"
    books:
      世界书条目名称: "[节庆_盟约日_文本_盟约日观礼手册]"
      书名: 盟约日观礼手册
      摘要内容: "要写东西就自行填上内容"
      关键字:
        - 盟约
        - 观礼
      次要关键字1:                                      // 示范支持第二组关键字
        逻辑: 与任意
        关键字:
          - AA
          - SS
      次要关键字2:                                      // 示范支持第三组关键字
        逻辑: 与任意
        关键字:
          - XX
          - YY
```

## 世界书编辑流程

推荐维护顺序：

1. 在世界书里新建或修改索引条目 [`[节庆_索引]`]，并按上方填写节庆节点
2. 世界书新增对应正文条目：
   - `[节庆_XX_介绍]`
   - `[节庆_XX_文本]`
3. 世界书条目的绿灯触发词是世界书条目自身名字，例如[节庆_盟约日_介绍]的绿灯触发词是`[节庆_盟约日_介绍]`
4. 重新导入或同步世界书，再让脚本重新扫描
